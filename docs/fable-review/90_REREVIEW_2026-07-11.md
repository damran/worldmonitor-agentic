## 1. Confirmed findings (15)

Grouped by the work-package that fixes them. Severity / effort in brackets.

### 1a. Calibration blockers — gate the G-B/real-seed milestone (→ Phase III below)

1. **The eval harness is not executable end-to-end** [med / days]. Nothing in `src/` builds
   the `Partition` mapping `resolution/eval.py` consumes, and nothing reads `er_gold_pair`
   back (it is only written — `gold.py`, `silver.py` — and backed up). "Run the harness on a
   real corpus" (50 §F3.1, G-B) is aspirational until a partition-builder + gold-reader
   exists. This is the highest-leverage fix in this review: without it the real-seed run
   produces data nothing can score.
2. **`silver.build_silver_pairs` is an unguarded pure-Python O(n²) loop** [med / days]
   (`resolution/silver.py:239-240`, no anchor pre-filter at :219). Infeasible at the
   OpenSanctions-scale corpus the real-seed plan targets; needs blocked pre-filtering (group
   by anchor key/value) or a hard input cap that fails loud.

### 1b. Test-estate integrity (→ H1 gate)

3. **`tests/property/test_prop_statement_spine.py::test_p_stmt_1_lossless_projection` runs a
   real Postgres testcontainer under `pytest -m "not integration"`** [high / hours] —
   violates the repo's own marker contract (pyproject `integration` marker docs +
   `tests/conftest.py`'s "the default (unit) run never touches Docker").
4. **The same test builds `make_engine()` inside the `@given` body with no per-example
   dispose** [med / hours] — the exact connection-leak anti-pattern already fixed in sibling
   files (QueuePool of real connections per Hypothesis example; masks RED-path assertions
   with "too many clients").

### 1c. Ops & deploy hygiene (→ H2/H3 gates)

5. **`.env.example` omits ~24 `settings.py` fields** [high / hours], including **two that
   unconditionally halt a production boot** (`SESSION_SECRET_KEY`, `ZITADEL_CLIENT_SECRET` —
   `validate_production_secrets`, settings.py:423-429) and the entire LLM-gateway block.
   First host deploy (S4) would hit an undocumented fail-closed boot. Fix + add a
   settings-vs-example drift test.
6. **Neo4j Browser (7474) and MinIO Console (9001) publish on 0.0.0.0** [med / hours] while
   every sibling data port is loopback-pinned (compose.yaml L65/L90 vs L24/66/89/105).
7. **Secret-hygiene asymmetry** [med / hours]: foundational credentials are plain `str`
   (settings.py:44/47/66/67/71/315) while newer ones are `SecretStr` with explicit
   never-echoes rationale (settings.py:256/281/296). Backfill the older ones.
8. **Container images are tag-pinned only, never digest-pinned** [med / hours] — the 50 §F8
   pin+hash ask is fully open and on no gate sequence (`@sha256` grep across deploy/ +
   Dockerfile: zero hits).
9. **No connector-staleness metric** [med / 1 gate]: `ConnectorInstance.last_run` is written
   on every finalize but never read by the metrics collector; only
   hard-disable-after-10-failures is observable. A silently dark source is invisible.
   (Converges with `91_OG_HARVEST_BACKLOG.md` item F-1 — one gate closes both.)
10. **Migrations runbook's primary procedure references a systemd unit that does not exist**
    [med-low / hours]; no compose-path redeploy-while-running runbook.

### 1d. Code correctness (→ H4 gate)

11. **`api/integrations.py` async POST handlers block the event loop** [med / hours]:
    `create_instance` (:244) and `_set_status` (:300, shared by enable/disable) are
    `async def` making sync SQLAlchemy calls on the loop thread — the same file already does
    it right (`run_instance` via `asyncio.to_thread`), and ADR 0092 §4 codifies the
    convention. (They `await request.form()`, so the fix is offloading the DB work, not
    de-asyncing.)

    on the queue table (db/models.py:73), single uniform insert site (ingest.py:214-220).
    Erasure cost currently grows with unrelated backlog.
