# Gate 1a ŌĆö Review-queue web UI (read-only surface) ŌĆö build spec

> Owning decision: **ADR 0103** (`docs/decisions/0103-review-queue-web-ui.md`). This spec is consumed
> **verbatim** by the test-author and the builder. Gate 1a is the **person-NEUTRAL, read-only** slice:
> promote the sign-off CLI's *view* to a server-rendered HTMX surface. It writes nothing ŌĆö no graph, no
> gold, no judgement, no merge decision. The person-affecting verdict + gold surface is **Gate 1b**,
> specified in ┬¦1B below (self-contained, so the 1b pass is cheap).
>

## 0. Goal (one paragraph)

Render the existing `merge_audit.decision == 'pending_review'` queue in the browser: `GET /review`
(queue list with counts, a base "blocked pending human sign-off" state, a prominent **sensitive** badge,
a **confidence band** styled from the merge score ŌĆö never a verdict, and the guard reason shown
verbatim) + `GET /review/card?canonical_id=ŌĆ”` (an HTMX fragment with side-by-side candidate cards built
from `ErQueueItem.raw_entity`, a **statement-level evidence diff**, and per-member source chips). Every
route is `get_principal`-gated. Vendor **htmx v2 (0BSD)** into `static/` (no CDN ŌĆö zero-egress). Truth-up
the CLI's mislabeled `reject` help. **No POST, no write, in this slice.**

## 2. Data facts the builder must honour (verified at `e227334`)

1. **A parked merge has NO `StatementRecord` rows.** The Gate 2a dual-write fires only on the promoted
   `"merged"` path (`pipeline.py:476-478`); the park path (`pipeline.py:426`) writes none. **The
   evidence diff MUST be built from `ErQueueItem.raw_entity`** (parse with
   `worldmonitor.ontology.ftm.make_entity`), never from the statement spine ŌĆö a `StatementRecord` query
   for a queued item is always empty and would falsely render "no evidence".
2. **`list_parked(session, neo4j)`** (`resolution/signoff.py:101`) already returns the queue as
   `ParkedMerge(canonical_id, source_ids, score, reason, graph_written)`. Reuse it verbatim; pass the
   read-only `neo4j` client so `graph_written` (the B-1 incomplete-sign-off recovery flag) is surfaced.
   It performs one Neo4j existence read per row (`execute_read`) ŌĆö acceptable for the small queue; do
   not change it.
3. **Do NOT call `signoff._member_rows` or `signoff._outbound_edges` from the read UI** ŌĆö they call
   `_dead_letter_poison` (`session.add`), i.e. they **write**. 1a needs its own read-only member loader
   that, on an unparseable `raw_entity`, **skips and marks the card "unparseable"** and writes nothing.
4. **Sensitivity badge** = `guard.sensitivity.is_sensitive(make_entity(m.raw_entity))` over the members
   (pure, deny-by-default, ADR 0047). **Do not** substring-match `merge_audit.reason` for the badge
   (the guard's own code warns the reason embeds hostile data-bearing fields). Show the reason verbatim
   (autoescaped) as "why parked".
5. **Confidence band** = `merge_audit.score` (a float; `ParkedMerge.score`) rendered as a
   gradient/bar + the numeric value, styled as a band ŌĆö **never** a pass/fail / "MATCH" verdict (C5,
   `docs/70` ┬¦10).
6. **Canonical id is a query param, not a path segment** (`GET /review/card?canonical_id=ŌĆ”`) ŌĆö ids can
   contain `:` (`qid:Q42`, `lei:ŌĆ”`). A `canonical_id` with no current `pending_review` audit ŌåÆ **404**.
7. **Auth/CSRF seam** to copy: `api/integrations.py` (`_csrf_token`, `get_principal`-gating,
   autoescape, `templates.TemplateResponse(request, name, context)`), `api/main.py` (`create_app`
   wiring), `api/deps.py` (`get_db`, `get_principal`, `get_neo4j`), `api/middleware.py` (302 for
   unauthenticated `text/html`, 401 for tokenless API).

## 3. Invariants for the test-author (name ┬Ę statement ┬Ę generator ┬Ę oracle)

> `@given` where meaningful (read-only + escaping); example tests for the rest. Never weaken an oracle
> (the `test-strictness` hook blocks net-removed asserts / added skips).

**INV-READONLY (`test_prop_review_readonly.py`, `@given`).**
- *Statement:* for any set of parked merges, `GET /review` and `GET /review/card` issue **zero writes**
  ŌĆö no graph write, no Postgres INSERT/UPDATE/DELETE.
- *Generator:* Hypothesis-build a list of `MergeAudit(decision='pending_review')` + matching
  `ErQueueItem(status='pending_review')` rows with varied member counts, scores in `[0,1]`, and
  reasons/property values drawn including hostile strings (`<script>`, quotes, unicode separators).
- *Oracle:* inject a **read-only Neo4j spy** whose `execute_read` returns configured rows and whose
  `execute_write` raises `AssertionError`; inject a session/sessionmaker whose `commit`/`flush` with a
  non-empty `dirty`/`new`/`deleted` set raises. Both GET routes return 200 and trigger neither.

**INV-XSS (`test_prop_review_readonly.py`, `@given`; and an example in `test_review_ui.py`).**
- *Statement:* a hostile `merge_audit.reason` or a hostile FtM property value is HTML-escaped by Jinja
  autoescape ŌĆö never rendered raw.
- *Oracle:* `"<script>alert(1)</script>"` in a reason and in a member property appears in the response
  as `&lt;script&gt;` and never as the raw tag. No `| safe` on untrusted data.

**INV-QUEUE (`test_review_ui.py`, example).**
- `GET /review` lists every `pending_review` merge (canonical id, member count, guard reason, score),
  shows a total count, and shows the base "blocked pending human sign-off" state on each.

**INV-SENSITIVE (`test_review_ui.py`, example).**
- A parked merge with a member carrying an FtM risk topic (e.g. `topics: ["sanction"]`, or an
  off-ontology code) renders the **prominent sensitive badge**; a benign merge does not. The badge is
  driven by `is_sensitive`, not by the reason text (assert a benign merge whose *reason* contains the
  word "sensitive" does **not** get the badge).

**INV-BAND-NOT-VERDICT (`test_review_ui.py`, example).**
- The score renders inside a `.wm-confidence-band` element with the numeric value; the response
  contains **no** verdict word for the band (no "MATCH"/"PASS"/"FAIL"/"VERDICT" styling on the score).

**INV-CARD-DIFF (`test_review_ui.py`, example).**
- `GET /review/card?canonical_id=<parked>` renders one card per member from `raw_entity` (schema + FtM
  props + a source chip with `source_id`/reliability/`retrieved_at`/raw pointer), and a diff block that,
  for a property where members **agree**, marks agreement, and for a property where members **differ**,
  marks a contradiction. An unparseable `raw_entity` renders an "unparseable" card and **no** write
  occurs (assert via the write spy). An unknown `canonical_id` ŌåÆ 404.

**INV-RECOVER (`test_review_ui.py`, example).**
- When the read-only Neo4j spy reports a node exists for a member/canonical id, the queue row surfaces
  the `graph_written` recovery flag (mirrors the CLI's "[GRAPH-WRITTENŌĆ”]" note).

**INV-AUTH (`test_review_ui.py`, example).**
- No `Authorization` header + `Accept: text/html` ŌåÆ 302 `/login`; tokenless API (`Accept:
  application/json`) ŌåÆ 401; a bad bearer ŌåÆ 401. (Reuse the `integrations` auth-gate pattern.)

**INV-VENDOR (`test_vendored_assets.py`, example).**
- `api/static/vendor/htmx.min.js` exists and is non-trivial (> 5 KB); `review.html` references only
  `/static/...` assets and contains **no** external host (`http://`, `https://`, `unpkg`, `jsdelivr`,
  `cdn`). `PROVENANCE.md` records a version whose major is `2`, license `0BSD`, and a sha256 that
  **matches** `sha256(htmx.min.js)`.

# ┬¦1B ŌĆö Gate 1b roadmap (verdict actions + gold hook; person_affecting: TRUE)

### ┬¦1B.1 New domain (all ADDITIVE ŌĆö no migration)

- **`signoff.abstain(session, *, canonical_id, approver, reason)`** ŌĆö sibling of approve/reject; writes
  one `SignOff(decision="abstained")` (fits `String(16)`), **no** graph/gold/judgement; leaves
  `merge_audit.decision == 'pending_review'` (revisitable); refuses on an already-terminal audit.
- **`gold.record_verdict_gold(session, *, canonical_id, verdict)`** ŌĆö loads the `MergeAudit`, then per
  ADR 0103 Decision C: **approve** ŌćÆ all `C(k,2)` pairs `label="match"`, `source="signoff:approve"`;
  **reject & k==2** ŌćÆ one pair `label="non_match"`, `source="signoff:reject"`; **reject & k>2** ŌćÆ
  **nothing**; **abstain** ŌćÆ **nothing**. Reuses `persist_gold_pairs` (canonical ordering + `ON
  CONFLICT DO NOTHING`, idempotent); `clerical_score=None`. `source` fits `String(32)` ŌåÆ **no
  migration**. **Does NOT touch `_record_judgements`** (the judgement surface stays all-pairs-negative
  on any reject ŌĆö intentional divergence, ADR 0103 Decision C).

### ┬¦1B.2 New UI + files

- `api/review.py` ŌĆö add `_check_csrf` + `POST /review/verdict` (form `canonical_id`, `verdict Ōłł
  {approve,reject,abstain}`, `csrf_token`; CSRF-check ŌåÆ dispatch to `signoff.approve`/`reject`/`abstain`
  in a worker thread if it blocks; then `gold.record_verdict_gold`; 303 back to `/review`). Surface
  `SignOffError` as a user-visible flash, not a 500.
- `review_card.html` ŌĆö add the verdict form + `a`/`r`/`x` keydownŌåÆsubmit (vanilla JS, no Alpine).
- `review.html` ŌĆö sort recently-`SignOff('abstained')` items last (read-side; no new column).
- `resolution/signoff.py` (+`abstain`), `resolution/gold.py` (+`record_verdict_gold`) ŌĆö additive.
- Tests: `tests/property/test_prop_review_verdict_gold.py`, `tests/unit/test_review_verdict.py`,
  `tests/integration/test_review_verdict.py`.

### ┬¦1B.3 Property obligations (mandatory `@given` ŌĆö merge/sign-off + gold invariant)

- **P-GOLD-FAITHFUL** ŌĆö over `@given` cluster size `kŌłł[2,N]` ├Ś verdict: the gold rows written are
  **exactly** the ADR 0103 Decision-C table (approve ŌćÆ the `C(k,2)` `match` pairs and nothing else;
  abstain ŌćÆ Ōłģ; reject k==2 ŌćÆ the one `non_match`; reject k>2 ŌćÆ Ōłģ). Oracle asserts the exact set of
  `(left_id,right_id,label,source)` written, canonical ordering, and `left_id <= right_id`.
- **P-VERDICT-IDEMPOTENT** ŌĆö re-POST of the same verdict converges: the person-affecting state
  (`merge_audit.decision`, signoff-written graph nodes, resolver judgements, gold rows) is identical
  after 1 vs N POSTs. Exclude the append-only `SignOff` audit-row count from the oracle (abstain's trail
  is append-only by design). Mirror `tests/property/test_b1_signoff_idempotency.py`.
- **P-GUARD-NOBYPASS** ŌĆö no UI route writes the graph except through frozen `signoff.approve`/`reject`;
  every state-changing route is CSRF-gated (absent/wrong token ŌåÆ 403 and **no** write). Reject/approve
  never bypass the orphan/state-machine guards.

### ┬¦1B.4 1b acceptance

All ┬¦5 criteria plus: P-GOLD-FAITHFUL / P-VERDICT-IDEMPOTENT / P-GUARD-NOBYPASS green `@given`;
integration approveŌćÆgraph+`match` gold, reject k==2ŌćÆmembers+`non_match` gold, reject k>2ŌćÆmembers+**no**
gold, abstainŌćÆ**no** gold and item stays in queue; `_record_judgements` behaviour byte-unchanged;
`db/migrations/**` untouched (assert no new migration file). ADR 0103 cosign stamped before merge.
